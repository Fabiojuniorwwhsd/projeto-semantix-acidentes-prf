# 05 — Checklist do dashboard no Looker Studio

## Objetivo do dashboard
Comunicar, de forma executiva e clara, os principais padrões dos acidentes em rodovias federais e apoiar decisões preventivas.

## Página 1 — Visão executiva
### KPIs sugeridos
- Total de acidentes
- Total de acidentes severos
- % de acidentes severos
- Mortos
- Feridos
- UF líder em acidentes severos

### Gráficos sugeridos
- Série temporal mensal de acidentes
- Barras por UF
- Barras por tipo de acidente
- Barras por causa presumida
- Heatmap ou tabela por dia da semana x faixa horária

### Filtros sugeridos
- Ano
- UF
- Município
- Tipo de acidente
- Gravidade
- Faixa horária

## Página 2 — Geografia e concentração
- Mapa por UF ou município
- Ranking de municípios/UFs
- Comparação acidentes x população (se aplicável)

## Página 3 — Perfil e severidade
- Distribuição de severidade
- Comparativo por fase do dia
- Comparativo por condição meteorológica
- Comparativo por traçado da via

## Boas práticas
- Título claro por gráfico
- Destaque para 3 a 5 KPIs
- Paleta simples e consistente
- Evitar excesso de visuais
- Inserir box com interpretações
- Colocar fonte dos dados no rodapé

## Hyperlinks recomendados
- Link para o portal oficial da PRF
- Link para o IBGE
- Link para o repositório GitHub do projeto

## O que o dashboard precisa responder
- Onde estão os acidentes mais severos?
- Quando eles acontecem com maior frequência?
- Quais fatores aparecem com mais força?
- Que ações preventivas parecem mais urgentes?
